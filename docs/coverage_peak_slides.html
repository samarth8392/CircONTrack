CircONTrack Coverage Peaks

Detecting statistically significant peaks in sequencing coverage

The Goal

Find "mountains" in sequencing read coverage that stand out above normal variation.

Core Idea

  • Genome split into sliding windows
  • Measure coverage per window
  • Model background variation
  • Find statistically enriched windows (peaks)

Step 1: Input

  • BAM file (aligned ONT reads)
  • Optional reference FASTA (chromosome lengths)

Step 2: Sliding Windows

Default: 1000 bp window, step 500 bp

For each window: count reads, sum coverage

Step 3: Background Model

Coverage is noisy → Negative Binomial fits better than Poisson.

Variance > Mean → Negative Binomial
Variance ≤ Mean → Poisson

Step 4: Statistical Testing

For each window: probability of seeing this coverage by chance

  • Negative Binomial survival function
  • One-tailed test for enrichment

Step 5: Multiple Testing

Use Benjamini-Hochberg FDR correction to avoid false positives.

Step 6: Biological Filters

  • Min fold change
  • Min coverage

Step 7: Merge Peaks

Combine peaks within a given merge distance

Output

  • BED file with peak coordinates
  • Optional plots showing coverage, p-values, fold-change
